## Vision

Build a blue-team research platform that deploys controlled clone tokens on devnet/testnet
and mainnet (bait tokens), observes how they appear in real token feeds, extracts detection
signatures, trains ML classifiers, and publishes the first public labeled clone detection
dataset (CloneNet). Source of truth: AGENTS.md.

### 10x Check

The real product is not the infrastructure. It is the **labeled dataset**. No public
clone-specific classifier or dataset exists. A "CloneNet" dataset (like ImageNet for
clone tokens) would become the foundation everyone else builds detectors on. The platform
generates the data. The dataset IS the moat.

### Dream State (12-month)

Open-source clone detection engine used by DEX aggregators, wallets, and exchanges.
Labeled dataset published on HuggingFace. Real-time detection API with <500ms latency.
Multi-chain coverage (Solana, Base, BNB Chain).

## Module Details

### RECON - Reconnaissance & Signal Intelligence

**Purpose:** Monitor token feeds across chains, detect new deployments, feed data to classifiers.

**Data sources (priority order):**
1. Geyser/Yellowstone gRPC (Solana, sub-second latency)
2. Helius DAS API (bulk token metadata)
3. DexScreener / Birdeye / GeckoTerminal APIs (5-10s polling)
4. Alchemy/Infura pending transactions (Base, BNB Chain)

**Signal scoring (0-100):**
- Name/ symbol similarity to known tokens: 0.30
- Deployment timing relative to original: 0.20
- Metadata pattern match: 0.15
- Holder distribution pattern: 0.15
- Trading velocity: 0.10
- ORACLE prediction confidence: 0.10

**Channels:**
- `recon:signals` - New token detections with composite scores
- `recon:raw` - Raw token feed for all new deployments
- `recon:graduations` - Pump.fun graduation events

### MINT - Clone Deployment Engine (Research Only)

**Purpose:** Deploy controlled, labeled clone tokens on devnet/testnet for training data.

**Constraints:**
- Maximum 50 clones per day across all networks (RISK enforced)
- All clones tagged with research metadata (provenance, original token, timestamp)
- Clone metadata includes `X-Research-Tag: CloneNet-Research` in URI

**Deployment pipeline (5 stages, <5 seconds):**
1. Metadata generation (name variant, ticker, description with controlled variation)
2. Contract preparation (SPL Token or Token-2022 template)
3. Deployment via TXENG with priority fees
4. Post-deployment hardening (mint authority revoked, freeze authority revoked)
5. Auto-labeling (generate CloneNet record with ground-truth labels)

**Naming strategies for research:**
- Character substitution (0 for O, l for I)
- Homophones
- Suffix addition
- Unicode lookalikes
- Each strategy gets its own label in the dataset

### DETECT - Classification Engine

**Purpose:** Classify tokens as clone/original/unknown using rule-based + ML + zero-shot ensemble.

**Classification methods:**

1. **Rule-based (Phase 1):**
   - Levenshtein distance on name/symbol vs known originals
   - Metadata image hash comparison
   - Deployment timing analysis (within N seconds of original)
   - Creator wallet pattern matching

2. **ML classifier (Phase 2):**
   - Features: name_similarity, symbol_similarity, deployment_delay_sec,
     holder_count_at_1min, holder_count_at_5min, buy_volume_ratio,
     creator_wallet_age, mint_authority_revoked, metadata_similarity
   - Model: XGBoost (fast training, good feature importance)
   - Training data: MINT-deployed clones (positive) + known originals (negative)
   - Retraining: daily from ClickHouse

3. **Zero-shot (ORACLE):**
   - LLM receives structured prompt with token metadata + early trading data
   - Returns clone_probability (0-1) + reasoning
   - Used for novel patterns the ML model hasn't seen

**Ensemble logic:**
```
if rule_based_score > 0.9: CLONE (high confidence)
elif ml_score > 0.8: CLONE (high confidence)
elif oracle_score > 0.8 AND ml_score > 0.6: CLONE (medium confidence)
elif ml_score < 0.3 AND oracle_score < 0.3: ORIGINAL
else: UNKNOWN (queue for human review)
```

**Output channels:**
- `detect:classifications` - Token classification results
- `detect:features` - Extracted features for ML training
- `detect:alerts` - High-confidence clone detections

### ORACLE - Zero-Shot Prediction

**Purpose:** LLM-based classification for tokens with no historical precedent.

**LLM integration:**
- Primary: GPT-4o (fast, cheap)
- Secondary: Claude (higher quality, slower)
- Fallback: Local model via Ollama if API unavailable

**Prompt structure:**
```
Given this token:
- Name: {name}
- Symbol: {symbol}
- Description: {description}
- Created: {timestamp}
- First 60s trading: {buy_count} buys, {unique_buyers} unique wallets
- Holder distribution: {top_10_pct}%
- Similar tokens found: {similar_tokens_json}

Is this token a clone/impersonator of another token?
Respond with JSON: {"clone_probability": 0.0-1.0, "original_token": "name or null", "reasoning": "..."}

Chain of thought:
1. Analyze name similarity to known tokens
2. Assess trading pattern (sniper bots, wash trading indicators)
3. Evaluate metadata authenticity
4. Consider deployment timing relative to similar tokens
5. Produce calibrated probability
```

**Cost control:**
- Only classify tokens with recon signal score > 60
- Cache results in Redis (TTL 5 minutes)
- Batch classifications (process 10 tokens per LLM call)
- Budget: max $10/day in LLM API costs (RISK enforced)

### TXENG - Transaction Engineering

**Purpose:** Handle transaction submission across multiple paths for research coverage.

**Capabilities:**
- Jito bundle submission (Solana devnet)
- Standard RPC submission
- Transaction simulation (pre-flight validation)
- Compute unit optimization
- Priority fee calculation

**Research value:**
- Test how clones appear when submitted via different transaction paths
- Measure detection latency across paths
- Validate that classifiers work regardless of submission method

### CHAIN - Multi-Chain Support

**Purpose:** Extend clone deployment and detection to Base and BNB Chain.

**Chain priority:**
| Chain | Priority | Testnet | RPC Provider |
|-------|----------|---------|-------------|
| Solana | P0 | Devnet | Helius + Triton One |
| Base | P0 | Sepolia | Alchemy |
| BNB Chain | P1 | Testnet | Infura |

**Cross-chain detection:**
- When a token trends on Solana, check Base and BNB for clones
- Cross-chain similarity scoring
- Unified CloneNet schema works across all chains

### RISK - Safety & Controls

**Purpose:** Prevent the research platform from being misused or going out of control.

**Hard limits:**
- Max 50 clone deployments per day across all networks
- Max $10/day LLM API budget
- All clones tagged with research provenance (internal DB only, never on-chain for mainnet bait)
- No wash trading patterns
- No anti-detect infrastructure
- Mainnet bait tokens must be indistinguishable from real clone tokens in on-chain metadata

**Circuit breakers:**
- Level 1 (Yellow): >30 clones/hour or >$5 LLM spend in 1 hour -> throttle to 10/hour
- Level 2 (Orange): >40 clones/hour or error rate >30% -> halt MINT, keep RECON running
- Level 3 (Red): Any mainnet deployment attempt or unauthorized operation -> full halt, alert

**Audit logging:**
- Every clone deployment logged with: timestamp, original token, clone address, naming strategy, operator
- All LLM API calls logged with prompt + response + cost
- Daily audit report generated from ClickHouse
